Everyone racing to make their site "AI-ready" is optimizing the one thing that matters least. The data says so. Almost nobody is acting on it.

Princeton's GEO study and the GEO-16 citation work landed on the same finding. Generative engines weight independent earned media and routinely drop vendor-owned pages. Your perfect llms.txt can still never get cited.

The citation map explains why. Fifteen domains capture 68% of AI citations. Wikipedia alone is roughly 48% of ChatGPT's top-10. Agents select from sources they already trust.

So there are two jobs, not one.

- **Stock your own shelf.** llms.txt, markdown mirrors, structured docs, clean JSON-LD.
- **Earn corroboration** from the sources the model already trusts.

Everyone is doing the first. Almost no one is doing the second.

**My read:** the shelf is necessary and not sufficient. If the trusted sources never mention you, "extractable" just means an agent can read a page it will never visit.

There is a real nuance here, and I want to be honest about it. A personal site like this one is less vendor than a corporate domain, but it is still self-published. It is not the same as third-party corroboration. It upgrades the named-operator surface. It does not replace getting cited by Wikipedia, the press, or a peer who has no stake in you.

The unclaimed move in agentic GTM is not better on-page markup. It is earned third-party presence, run and measured like a real channel. That is the gap I am watching.

(Source: Aggarwal et al., "GEO: Generative Engine Optimization," KDD 2024.)
